# Agent 编程任务调度台全面任务拆解

## 1. 拆解说明

本文档基于 `需求解析.md` 最终版拆解，目标是把产品需求转化为可执行、可排期、可验收的研发任务。

任务优先级说明：

| 优先级 | 含义 |
| --- | --- |
| P0 | MVP 必须完成，影响核心闭环 |
| P1 | MVP 建议完成，影响可用性和安全性 |
| P2 | 增强版能力，可在 MVP 后迭代 |

任务状态说明：

| 状态 | 含义 |
| --- | --- |
| pending | 待开始 |
| in_progress | 进行中 |
| done | 已完成 |
| blocked | 阻塞 |

### 1.1 当前进度基线

当前项目已完成到 S3 阶段：

1. S0 项目准备与规范：已完成。
2. S1 桌面端基础工程：已完成。
3. S2 本地数据与任务模型：已完成。
4. S3 Git Worktree 隔离工作区：已完成。

后续任务默认从 S4 命令执行与日志系统继续推进。若 S4 及之后能力依赖 S0-S3 的新增字段、配置项或 UI 入口，可以回补 S0-S3，但不重新推翻已完成成果。

### 17.1 亮点完整版执行顺序

由于当前目标是不考虑工期、优先保证最终结果有亮点，建议在 S4-S11 核心闭环跑通后，按以下顺序补齐差异化能力：

1. S12-E06 隐私证明账本，先保证数据流向可见、可审计。
2. S12-E07 运行契约，把模式、模型、思考强度、权限、预算和语言统一成任务启动边界。
3. S12-E09 不降智省 Token，建立上下文分层、Token 预算和质量兜底。
4. S12-E08 个人喜好蒸馏与个人工程画像，让 Agent 形成可解释、可确认的个人化。
5. S12-E10 内置 Skills、规则包与 Hooks Studio，让工程规范可沉淀、可扩展、可拦截。
6. S12-E05 差异化亮点能力，补齐证据包、质量门禁、风险雷达、任务胶囊、多模型对战和交付评分。

## 18. MVP 完成定义

MVP 被视为完成，必须满足：

1. 可以选择一个本地 Git 仓库。
2. 可以创建一个自然语言编程任务。
3. 系统可以为任务创建独立 Git Worktree。
4. Agent 可以生成 Todo List。
5. Agent 可以修改示例代码。
6. 系统可以执行验证命令并实时展示日志。
7. 验证失败时 Agent 可以进入自动修复循环。
8. 超过 5 轮失败后任务进入人工介入状态。
9. 验证成功后系统可以展示只读 Diff。
10. 系统可以展示测试报告和 Agent 交付说明。
11. 高风险命令会触发审批。
12. 用户确认后可以本地合入代码。
13. 合入冲突不会被静默覆盖。
14. API Key 不会出现在普通日志中。
15. 应用重启后可以看到历史任务和交付物。
16. 最近对话原文默认只保留 50 条，旧消息可生成滚动摘要。
17. 长期记忆可查看、编辑和删除。
18. 原始日志、截图、临时上下文和 Worktree 可按策略清理。
19. 清理后最终 Diff、测试报告摘要、审批记录和合入记录仍可查看。
20. SQLite 不保存大日志、大截图和完整仓库副本。
21. 每个完成任务都生成审计级证据包。
22. 合入前质量门禁默认启用，失败时不得默认合入。
23. 任务详情展示交付评分和风险解释。

### 18.1 亮点完整版完成定义

亮点完整版被视为完成，还必须满足：

1. 每个任务启动前都可以确认运行契约。
2. 每个任务都生成隐私账本，记录本地读取、模型上下文、脱敏、阻断和供应商信息。
3. 每次模型调用都能记录 Token 使用、上下文来源和是否触发质量兜底。
4. 用户可以查看、确认、编辑、拒绝候选偏好。
5. 用户可以创建和切换个人工程画像。
6. Agent 使用长期记忆、个人画像或偏好时必须在任务上下文中说明。
7. 内置 Skills 和规则包可以按作用域启用、禁用。
8. Hooks 可以在关键生命周期执行检查、阻断、追加上下文或触发审批。
9. 合入前交付审查页同时展示 Diff、测试、门禁、隐私、运行契约、Token 预算、风险和评分。
10. UI、Agent 输出、证据包、交付说明和提交信息支持多语言。
11. 新 UI 风格落地前有用户确认流程。
12. 亮点能力有端到端验收用例。
