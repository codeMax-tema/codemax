# S0-T01 MVP 范围清单

## MVP 必须实现

1. 本地 Git 仓库选择与合法性校验。
2. 自然语言任务创建。
3. Git Worktree 隔离工作区。
4. Agent Todo List。
5. Agent 代码修改。
6. 本地命令执行、实时日志捕获和日志落盘。
7. 自动验证与最多 5 轮自我修复。
8. 只读 Diff 展示。
9. 测试报告展示。
10. 用户确认后一键本地合入。
11. 高风险命令审批。
12. 模型供应商配置。
13. SQLite + 文件系统产物目录的分层存储。
14. 最近消息 + 滚动摘要 + 结构化长期记忆。
15. 日志、截图、临时上下文和 Worktree 的保留策略配置。
16. 审计级证据包 Proof Pack。
17. 合入前质量门禁 Quality Gate。
18. 交付评分 Delivery Score。

## MVP 暂不实现

1. 团队账号和权限体系。
2. 云端任务同步。
3. 插件市场。
4. 远程仓库自动推送。
5. 自动创建 Pull Request。
6. 多仓库跨项目编排。
7. 完整 IDE 编辑器能力。

## 完成定义

MVP 的核心闭环是：用户选择本地仓库，创建任务，系统创建 worktree，Agent 规划并修改代码，系统验证并展示证据，用户审查后决定是否本地合入。
