# S0-T03 高风险操作清单

## 风险等级

| 等级 | 含义 | 默认动作 |
| --- | --- | --- |
| high | 可能破坏代码、数据、系统或用户隐私 | 必须审批或禁止 |
| medium | 可能改变依赖、环境、Schema 或较大范围文件 | 默认审批 |
| low | 可重复、可回滚、影响范围清晰 | 可记录后执行 |

## 必须审批

| 操作 | 风险点 | 审批内容 |
| --- | --- | --- |
| 删除文件或目录 | 可能丢失用户代码或产物 | 路径、影响范围、恢复方式 |
| 修改数据库 Schema | 可能影响数据兼容 | 迁移内容、回滚方案 |
| 安装、升级或删除依赖 | 可能引入供应链和体积风险 | 包名、版本、原因 |
| 执行危险命令 | 可能破坏系统或仓库 | 命令、cwd、参数、理由 |
| 合入主代码库 | 影响用户主分支 | 最终 Diff、测试结果、提交信息 |
| 修改任务 Worktree 外路径 | 可能污染主工作区或系统目录 | 目标路径、必要性 |

## 默认禁止

- 远程推送、强推、修改远程配置。
- 格式化磁盘或系统级破坏命令。
- 写入用户主目录中与任务无关的位置。
- 在日志或交付物中输出未脱敏 API Key。

## 命令风险关键词

首版扫描至少覆盖：

- Windows：`del`、`rmdir`、`format`、`Remove-Item -Recurse`、`takeown`、`icacls`
- Unix-like：`rm -rf`、`sudo`、`chmod -R`、`chown -R`、`mkfs`
- Git：`push`、`push --force`、`reset --hard`、`clean -fd`
- Package：`npm install`、`pnpm add`、`pip install`、`cargo add`
