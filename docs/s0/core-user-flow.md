# S0-T02 核心用户流程

## 主流程

```text
选择本地 Git 仓库
  -> 校验仓库和主工作区状态
  -> 创建自然语言任务
  -> 创建任务分支和 Git Worktree
  -> Agent 分析任务并生成 Todo List
  -> Agent 在任务 Worktree 中修改代码
  -> 系统执行验证命令并捕获日志
  -> 验证失败时进入自动修复，最多 5 轮
  -> 验证成功后生成 Diff、测试报告、证据包和交付说明
  -> 用户审查 Diff、日志、报告、质量门禁和风险说明
  -> 用户确认后本地合入
```

## 审批流程

```text
Agent 申请高风险操作
  -> 系统扫描风险并创建审批请求
  -> 任务进入等待审批
  -> 用户查看操作内容、目录、风险等级、影响范围和 Agent 理由
  -> 用户选择允许、拒绝或要求修改
  -> 系统记录审批结果
  -> Agent 按审批结果继续或重新规划
```

## 失败处理

- 命令失败：保存 stdout、stderr、退出码、耗时和关键错误摘要。
- 自动修复失败超过 5 轮：任务进入“需要人工介入”。
- 合入冲突：停止合入，展示冲突文件，不得标记为成功。
- 应用重启：恢复任务列表、日志路径、审批记录、交付物索引和记忆摘要。
