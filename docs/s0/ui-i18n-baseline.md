# UI 与国际化基线

这是用户额外要求的 S0 落点：真正进入 UI 设计或页面实现前，必须先询问用户意见。

## UI 风格策略

- 默认风格：简约、工作台式、信息密度适中。
- 可切换风格：后续至少保留主题扩展点，例如简约、紧凑、深色。
- 设计重点：任务状态清晰、风险提示醒目、日志和 Diff 可扫描。
- 不做营销式首页，首屏应直接服务仓库选择、任务总览或当前任务。

## 设计确认规则

进入 S6 前端页面前，需要向用户确认：

1. 默认简约风格是否满足预期。
2. 是否需要更紧凑的专业模式。
3. 是否优先做浅色、深色，还是同时做。

## 国际化规则

- 默认语言：`zh-CN`。
- 回退语言：`en-US`。
- 所有用户可见文本使用 i18n key，不在组件里硬编码。
- 状态、风险等级、按钮、错误提示和门禁结果都需要进入翻译资源。

## 无障碍与可用性

- 状态不只依赖颜色，还要有文本和图标。
- 风险审批按钮必须明确，避免误触。
- 日志、大 Diff 和任务列表必须按需加载，避免卡顿。
